# APNG Generator — まばたきイン / まばたきアウト エフェクト追加 実装仕様書

---

## 1. 概要

APNG Generator に新規トランジション効果「**まばたきイン**」「**まばたきアウト**」を追加する。
上下のまぶた（黒い領域）が開閉することで、キャラクター視点の「目を開ける / 閉じる」演出を実現する。

### 対象ツール

- APNG Generator（https://apng-generator-tan.vercel.app/）

### 追加する効果

| 効果名 | カテゴリ | 概要 |
|--------|----------|------|
| まばたきイン | 登場 | まぶたが閉じた状態 → 開いて画像が表示される |
| まばたきアウト | 退場 | 画像が表示された状態 → まぶたが閉じて画面が黒くなる |

### 想定ユースケース

- ココフォリア等でのTRPGセッション演出
- 「目を覚ます」「意識を取り戻す」シーン（まばたきイン）
- 「気を失う」「眠りに落ちる」シーン（まばたきアウト）
- CoC系シナリオのSAN値喪失・気絶・目覚め演出

## 2. 効果オプション仕様

効果オプションは **2行構成**（既存効果の最大行数に準拠）。

### 2.1 オプションUI

```
瞬き:  [1回] [2回] [3回]
演出:  [なし] [ぼやけ] [揺れ] [ぼやけ+揺れ]
```

- 各行はボタングループ（既存の方向選択・サイズ選択と同じUI形式）
- デフォルト値：瞬き「1回」、演出「なし」

### 2.2 オプション定義

#### 瞬き（blinkCount）

1回のトランジション中に行う瞬き（まぶたの開閉）の回数。

| 選択肢 | 値 | 動作 |
|--------|----|------|
| 1回 | `1` | まぶたの微動を伴いながら1回で開く（or 閉じる）。最もシンプルな演出 |
| 2回 | `2` | 薄目 → パチッと閉じ → 開眼。少し不安定な目覚め感 |
| 3回 | `3` | パチパチパチ → やっと開く。朦朧・意識混濁状態の表現 |

**重要：** 瞬き「1回」（最小値）であっても、まぶたの開閉は完全にリニア（直線的）にはせず、微かな速度変化・揺らぎを持たせること（後述「ベース動作」参照）。

#### 演出（visualEffect）

まぶたの動きに加えて、画像（視界）側に付与する追加演出。

| 選択肢 | 値 | 効果 |
|--------|-----|------|
| なし | `none` | まぶたの動きのみ。追加の視覚効果なし |
| ぼやけ | `blur` | 画像にガウシアンブラーを適用。閉じているほどぼやけ、開くほどクリアになる |
| 揺れ | `shake` | 画像の描画位置を毎フレーム微小にオフセット。閉じているほど揺れが大きい |
| ぼやけ+揺れ | `blur_shake` | 上記2つを同時に適用。最も朦朧とした印象 |

## 3. アニメーション仕様

### 3.1 ベース動作（全パターン共通）

まぶたの開閉は常に**人間的な揺らぎ**を持つ。

- 開閉の進行速度は一定ではなく、イージング関数を適用する
- 完全に閉じた状態でも微かに開きかける→戻るような微動（tremor）を入れる
- これにより「機械的にスーッと開く」のではなく、「人間がまぶたを開けている」感覚を出す

### 3.2 まぶたの形状

まぶたは上下2つの黒い領域として描画する。

- 境界線の形状は**楕円弧**（目の形）にする
- 画面中央を水平に分割する直線ではなく、緩やかなカーブで分割する
- 上まぶた：画面上部から下方にかけた黒領域（下端が楕円弧）
- 下まぶた：画面下部から上方にかけた黒領域（上端が楕円弧）

```
完全閉じ（openRatio = 0）:
┌────────────────────┐
│████████████████████│
│████████████████████│
│████████████████████│  ← 全面黒
│████████████████████│
│████████████████████│
└────────────────────┘

中間（openRatio ≒ 0.4）:
┌────────────────────┐
│████████████████████│
│██████████████████ ██│
│█████╲          ╱████│
│       ╲  画像  ╱      │  ← 楕円弧の隙間から画像が見える
│████████╲____╱███████│
│████████████████████│
└────────────────────┘

完全開き（openRatio = 1）:
┌────────────────────┐
│                    │
│                    │
│      画像全体       │  ← まぶた領域なし
│                    │
│                    │
└────────────────────┘
```

### 3.3 openRatio タイムライン

`openRatio` は 0（完全閉じ）〜 1（完全開き）の値で、各フレームのまぶた開き具合を制御する。

#### まばたきイン（瞬き1回の場合）

```
時間 →  0%                                        100%
         |                                          |
openRatio: 0 ~~微動~~ → 0.05 ~~ イージング ~~→ 1.0
           [  微動  ] [      メイン開眼       ]
```

#### まばたきイン（瞬き2回の場合）

```
時間 →  0%                                              100%
         |                                                |
openRatio: 0 ~微動~ 0.4 → 0.05 → 0.5 ~~ イージング ~~→ 1.0
           [ 微動 ][ 薄目 ][ 閉じ ][ パチッ→完全開眼 ]
                    ~~ 瞬き1 ~~
```

#### まばたきイン（瞬き3回の場合）

```
時間 →  0%                                                        100%
         |                                                          |
openRatio: 0 ~微動~ 0.3→0.05 → 0.5→0.1 → 0.6 ~~ イージング ~~→ 1.0
           [ 微動 ][ 薄目→閉 ][ 開き→閉 ][ パチッ→完全開眼 ]
                   ~~ 瞬き1 ~~ ~~ 瞬き2 ~~
```

#### まばたきアウト

上記の**逆順**。openRatio が 1 → 0 に向かう。

### 3.4 演出パラメータ

#### ぼやけ（blur）

- `blurAmount = (1 - openRatio) * maxBlur`
- `maxBlur` = 6px（目安）
- Canvas API: `ctx.filter = 'blur(${blurAmount}px)'`
- openRatio が 0（完全閉じ）のとき最大ブラー、1（完全開き）のとき 0

#### 揺れ（shake）

- `shakeIntensity = (1 - openRatio) * maxShake`
- `maxShake` = 8px（目安）
- 毎フレーム、画像の描画オフセットにランダム値を加算
  - `offsetX = (Math.random() - 0.5) * 2 * shakeIntensity`
  - `offsetY = (Math.random() - 0.5) * 2 * shakeIntensity`
- openRatio が 0 のとき最大揺れ、1 のとき揺れなし

#### ぼやけ+揺れ（blur_shake）

- 上記2つを同時に適用

## 4. 実装ガイド

### 4.1 フレーム描画フロー（1フレームあたり）

```
1. Canvas クリア
2. openRatio を計算（タイムライン + 瞬きパターン + 微動）
3. 演出パラメータを計算（blur量、shake量）
4. ctx.filter にブラーを設定（blur / blur_shake の場合）
5. 画像を描画（shakeオフセット適用）
6. ctx.filter をリセット
7. まぶたマスク（黒い領域）を上に描画
```

### 4.2 openRatio 計算の実装方針

```javascript
function calcOpenRatio(progress, blinkCount) {
  // progress: 0.0 〜 1.0（アニメーション全体の進行度）
  // blinkCount: 1 | 2 | 3

  // --- タイムライン分割 ---
  // 瞬き回数に応じてフェーズを分割
  // 最終フェーズは常に「完全開眼へのイージング」

  // --- ベース微動 ---
  // 全フェーズにわたり、openRatioに微小なノイズを加算
  // tremor = Math.sin(progress * Math.PI * 高周波) * 微小振幅
  // ただし openRatio が 0 または 1 に十分近い場合はクランプ

  // --- イージング ---
  // 各フェーズの遷移にはeaseInOut系のイージングを適用
  // 推奨: easeInOutCubic または easeInOutQuad
}
```

### 4.3 まぶた描画の実装方針

```javascript
function drawEyelidMask(ctx, width, height, openRatio) {
  const centerY = height / 2;
  const maxGap = height * 0.6;   // まぶたが開ける最大幅
  const currentGap = maxGap * openRatio;

  ctx.fillStyle = '#000000';

  // --- 上まぶた ---
  ctx.beginPath();
  ctx.moveTo(0, 0);
  ctx.lineTo(width, 0);
  ctx.lineTo(width, centerY - currentGap / 2);
  // 楕円弧（下に膨らむカーブ = 上まぶたの下端）
  ctx.quadraticCurveTo(
    width / 2, centerY - currentGap / 2 + currentGap * 0.3,
    0, centerY - currentGap / 2
  );
  ctx.closePath();
  ctx.fill();

  // --- 下まぶた ---
  ctx.beginPath();
  ctx.moveTo(0, height);
  ctx.lineTo(width, height);
  ctx.lineTo(width, centerY + currentGap / 2);
  // 楕円弧（上に膨らむカーブ = 下まぶたの上端）
  ctx.quadraticCurveTo(
    width / 2, centerY + currentGap / 2 - currentGap * 0.3,
    0, centerY + currentGap / 2
  );
  ctx.closePath();
  ctx.fill();
}
```

## 5. まばたきイン / まばたきアウト の関係

2つの効果は**同一の描画ロジック**を共有し、時間軸の方向のみが異なる。

```
まばたきイン:   progress を 0→1 で処理（openRatio: 0→1）
まばたきアウト: progress を 1→0 で処理（openRatio: 1→0）
```

つまり `renderFrame` 関数は共通化し、`direction` パラメータで切り替える。

```javascript
function renderEyeFrame(ctx, img, width, height, progress, options, direction) {
  const effectiveProgress = direction === 'open' ? progress : 1 - progress;
  const openRatio = calcOpenRatio(effectiveProgress, options.blinkCount);
  // ... 以降共通
}
```

## 6. オプション組み合わせ一覧とプレビュー期待動作

| 瞬き | 演出 | 動作イメージ |
|------|------|-------------|
| 1回 | なし | まぶたが微動しながら1回で開く。最もシンプル |
| 1回 | ぼやけ | 開くにつれ視界がクリアに。穏やかな目覚め |
| 1回 | 揺れ | 開くにつれ視界が安定。軽い衝撃後の回復 |
| 1回 | ぼやけ+揺れ | ぼやけた視界が揺れながらクリアに。軽い朦朧状態 |
| 2回 | なし | 薄目→パチッ→開眼。少し不安定な目覚め |
| 2回 | ぼやけ | 不安定な目覚め + ぼやけ。夢から覚める演出 |
| 2回 | 揺れ | 不安定な目覚め + 揺れ。揺さぶられて起きる |
| 2回 | ぼやけ+揺れ | かなり朦朧とした状態からの覚醒 |
| 3回 | なし | パチパチパチ→やっと開く。意識混濁 |
| 3回 | ぼやけ | 意識混濁 + ぼやけ。薬物・催眠からの覚醒 |
| 3回 | 揺れ | 意識混濁 + 揺れ。殴られた後、SAN直葬後 |
| 3回 | ぼやけ+揺れ | 最もヘビー。完全な朦朧状態からの覚醒 |

## 7. マニュアル / 効果カタログ向け情報

### 効果カタログ掲載テキスト案

#### まばたきイン

> キャラクター視点で「目を開ける」演出。上下のまぶたが開き、画像が現れます。
> 瞬きの回数や、ぼやけ・揺れの演出を組み合わせることで、穏やかな目覚めから朦朧とした意識回復まで幅広いシーンに対応します。

#### まばたきアウト

> キャラクター視点で「目を閉じる」演出。上下のまぶたが閉じ、画面が暗転します。
> 気絶、眠り、意識喪失などのシーンに最適です。

### ココフォリアでの推奨用途

| 用途 | ループ設定 | 推奨設定例 |
|------|-----------|-----------|
| 立ち絵（登場演出） | OFF | まばたきイン / 瞬き1回 / ぼやけ |
| 前景/背景（場面転換） | OFF | まばたきイン / 瞬き2回 / ぼやけ+揺れ |
| マーカーパネル | OFF | まばたきアウト / 瞬き1回 / なし |
| カットイン | ON | 非推奨（ループ向きの演出ではない） |
